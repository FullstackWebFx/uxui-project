Модуль 1

Основи UX: дослідження та структури

40 год • research • CJM • personas ▾

Коротко: цей модуль навчає методам збору даних про користувача, синтезу та формуванню інсайтів, що ляжуть в основу подальших дизайн-рішень. Ви навчитесь проводити інтерв'ю, аналізувати відповіді, будувати personas та customer journey maps (CJM).

Покроковий план (по тижнях)

  1. Тиждень 1: UX fundamentals, double diamond, research ethics.
  2. Тиждень 2: Підготовка та проведення інтерв'ю; створення гайдів інтерв'ю.
  3. Тиждень 3: Синтез: affinity mapping, clustering, insight statements.
  4. Тиждень 4: Personas, CJM, task flows, presentation of findings.

Практичне завдання

  • Провести 3 інтерв'ю, записати нотатки та короткі транскрипти.
  • Зробити affinity map і виділити 3 інсайти.
  • Підготувати 1 persona та CJM з pain/gain та action items.

Learning outcomes

  • Навички проведення qualitative research.
  • Уміння синтезувати дані в дієві гіпотези.
  • Підготовка deliverables, придатних для презентації продуктовим командам.

Rubric оцінювання (зразок)

Research quality
Interview completeness, quality запитань, depth of probes. 0–5
Synthesis & insights
Кількість/якість insights, їх логічність та actionability. 0–5
Deliverables
Чіткий CJM, реалізовані recommendations. 0–5

Поради від наставника

  • Не намагайтесь охопити все в 1 інтерв'ю — краще глибоко в 3–5.
  • Пишіть notes у форматі: observation → quote → implication.

Модуль 2

Interaction Design та Wireframing

45 год • wireframes • patterns ▾

Коротко: тут ми вчимося переводити інсайти в flows і wireframes — від початкового скетчу до mid-fi та handoff'у розробнику.

Теми по заняттях

  1. Sketching & idea exploration (paper first)
  2. Low-fi wireframes: framing and layout
  3. Mid-fi: spacing, alignment, information density
  4. Interaction specs and microcopy

Практика

Завдання: редизайн checkout flow — deliverables: low-fi set, clickable mid-fi прототип, interaction spec (state table).

Deliverables checklist

  • Flow diagram (user steps)
  • Low-fi wireframes for core screens
  • Mid-fi prototype with basic interactions
  • Interaction specification: states, errors, success

Кращі практики

  • Починайте з головного сценарію — не всі edge-cases відразу.
  • Прокручуйте interaction spec: що має відбутися при помилці?
Інструменти & assets

Модуль 3

UI: візуальний дизайн та компоненти

60 год • design system • tokens ▾

Коротко: створюємо дизайн-систему: від типографіки до компонентів та документації. У результаті — працююча бібліотека компонентів та guideline для командної роботи.

Ключові теми

  • Type scale & hierarchy; responsive typography
  • Color systems: semantic tokens
  • Component architecture: atoms -> molecules -> organisms
  • Design tokens → CSS variables & tokens export

Практика

Deliverable: стилгайд + бібліотека компонентів (12+ компонентів) + usage examples.

Rubric

Consistency
Відступи/радіуси/тіні однакові. 0–5
Reusability
Компоненти використовуються в декількох сценаріях. 0–5
Accessibility
Контраст, focus states, keyboard navigation. 0–5
Швидкі поради
  • Документуйте не тільки вигляд, а й поведінку компоненту.
  • Створюйте tokens як джерело істини для розробки.

Модуль 4

Прототипування і тестування

40 год • testing • prototypes ▾

Коротко: навчимо створювати прототипи, проводити користувацьке тестування (moderated / unmoderated), збирати та аналізувати дані, робити пріоритизацію фіксів.

Що ви вивчите

  • Види прототипів, інструменти та trade-offs
  • Планування usability test: tasks, metrics
  • Moderated vs. unmoderated, recruitment, ethical considerations

Практика

Кожна група проводить 5 moderated tests, готує report з findings, priority fixes та before/after recommendations.

Deliverables

  • Test plan & recruitment script
  • Raw recordings + summary spreadsheet
  • Priority fixes + rationale
Інструменти
Lookback / Maze

Figma prototypes

Модуль 5

Frontend для дизайнерів

35 год • HTML/CSS/JS • design→code ▾

Коротко: практичні навички перекладу дизайну у валідвний код: семантичний HTML, responsive CSS (Flexbox, Grid), design tokens → CSS variables, невеликі JS-interactions.

План

  1. Semantic HTML & aria basics
  2. CSS layout: Flex & Grid
  3. Responsive patterns & mobile-first
  4. Minimal JS interactions: tabs, modal, form validation

Практика

Ви реалізуєте адаптивну компонент-картку з вкладками та простим JS, підкреслюючи доступність.

Deliverables

  • HTML prototype
  • CSS module з variables
  • Documentation: how to use component

Модуль 6

Кар'єрна підготовка та проекти

50 год • portfolio • mock interviews ▾

Коротко: підготовка portfolio-ready кейсу, презентація, mock interviews, оптимізація LinkedIn та підготовка до технічних співбесід.

Теми

  • Case-study structure: context → process → solution → impact
  • Storytelling & storyboarding for demo
  • Mock interviews: portfolio & behavioral questions
  • LinkedIn, резюме, personal brand

Практика

Фінальний проект: complete case-study + презентація 10–12 хв. Кожен студент проходить mock interview з фідбеком.

Checklist

  • Чітко визначена роль у проєкті
  • Процес документовано з доказами (скріни, метрики)
  • Опис внеску та impact (числа/результати)

Career kit (deliverables)

  • Готовий case-study для portfolio
  • Сторінка проекту або презентація
  • Сценарії для mock-interview
Швидкі поради
  • Подавайте результати у вигляді impact: «до» / «після».
  • Тренуйте презентацію, записуйте на відео — це допомагає в оцінці delivery.
